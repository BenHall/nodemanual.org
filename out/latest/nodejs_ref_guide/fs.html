In Node.js, file input and output is provided by simple wrappers around standard POSIX functions. All the read and write methods methods have asynchronous and synchronous forms. To use this module, include require('fs') in your code.

The asynchronous form always take a completion callback as its last argument. The arguments passed to the completion callback depend on the method, but the first argument is always reserved for an exception. If the operation was completed successfully, then the first argument will be null or undefined.

When using the synchronous form, any exceptions are immediately thrown. You can use try/catch to handle exceptions, or allow them to bubble up.

In busy processes, the programmer is strongly encouraged to use the asynchronous versions of these calls. The synchronous versions block the entire process until they complete—halting all connections. However, with the asynchronous methods, there is no guaranteed ordering.

When processing files, relative paths to filename can be used; however, this path is relative to process.cwd().

Here's an example of the wrong way to perform more than one asynchronous operation: 

fs.rename('/tmp/hello', '/tmp/world', function (err) {
  if (err) throw err;
  console.log('renamed complete');
});
    // ERROR: THIS IS NOT CORRECT!
fs.stat('/tmp/world', function (err, stats) {
  if (err) throw err;
  console.log('stats: ' + JSON.stringify(stats));
});

In the example above, it could be that fs.stat is executed before fs.rename. The correct way to do this is to chain the callbacks, like this: 

fs.rename('/tmp/hello', '/tmp/world', function (err) {
  if (err) throw err;
  fs.stat('/tmp/world', function (err, stats) {
    if (err) throw err;
    console.log('stats: ' + JSON.stringify(stats));
  });
});

                                                                                                                Required. The filename (or directory) to watch

                                                                                                                options

                                                                                                                Optional. An optional arguments indicating how to watch the files

                                                                                                                listener

                                                                                                                Function

                                                                                                                Required. The callback to execute each time the file is accessed

                                                                                                                event

                                                                                                                String

                                                                                                                Required. Either 'rename' or 'change'

                                                                                                                filename

                                                                                                                String

                                                                                                                Required. The name of the file which triggered the event

                                                                                                                Description

                                                                                                                options, if provided, should be an object containing a boolean member persistent, which indicates whether the process should continue to run as long as files are being watched.

                                                                                                                Providing the filename argument in the callback is not supported on every platform (currently it's only supported on Linux and Windows). Even on supported platforms, filename is not always guaranteed to be provided. Therefore, don't assume that filename argument is always provided in the callback, and have some fallback logic if it is null, like in the provided example.

                                                                                                                Example

                                                                                                                fs.watch('somedir', function (event, filename) {
    console.log('event is: ' + event);
    if (filename) {
        console.log('filename provided: ' + filename);
    } else {
    console.log('filename not provided');
    }
});
