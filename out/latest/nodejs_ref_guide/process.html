The process object is a global object, and can be accessed from anywhere. It is an instance of EventEmitter.

Example: Handling Signal Events

Signal events are emitted when processes receive a signal. See sigaction(2) for a list of standard POSIX signal names such as SIGINT, SIGUSR1, etc.

Events

    • exit()

    Emitted when the process is about to exit. This is a good hook to perform constant time checks of the module's stat...

    Emitted when the process is about to exit. This is a good hook to perform constant time checks of the module's state (like for unit tests). The main event loop will no longer be run after the exit callback finishes, so timers may not be scheduled.

       

    Example: Listening for an 'exit' event:

    process.on('exit', function () {
    process.nextTick(function () {
        console.log('This will not run');
    });
    console.log('About to exit.');
});
      • uncaughtException(Error err)

      Emitted when an exception bubbles all the way back to the event loop. If a listener is added for this exception, the...

      Emitted when an exception bubbles all the way back to the event loop. If a listener is added for this exception, the default action (which is to print a stack trace and exit) won't occur.

         

      Arguments

      NameTypeDescription

      err

      Error

      Required. The standard Error Object

      Example: Listening for an 'uncaughtException':

      An uncaughtException is a very crude mechanism for exception handling. Using try / catch in your program gives you more control over your program's flow. Especially for server programs that are designed to stay running forever, uncaughtException can be a useful safety mechanism.

                  NameTypeDescription
